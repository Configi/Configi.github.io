Module opkg

Ensure that a package managed by Opkg is installed or absent.

Info:

  • License: MIT http://opensource.org/licenses/MIT
  • Author: Eduardo Tongson propolice@gmail.com
  • Added: 0.9.0

Functions

present(force_depends, force_reinstall, force_overwrite, force_downgrade, force_maintainer, nodeps, proxy, update)
Install a package. See opkg --help

Subject:

    package

Aliases:

  • installed
  • install

Parameters:

  • force_depends install despite failed dependencies [CHOICES: "yes","no"]
  • force_reinstall reinstall package [CHOICES: "yes","no"]
  • force_overwrite overwrite files from other packages [CHOICES: "yes","no"]
  • force_downgrade Allow downgrading packages [CHOICES: "yes","no"]
  • force_maintainer Overwrite existing config files [CHOICES: "yes","no"]
  • nodeps Do not install dependencies [CHOICES: "yes","no"]
  • proxy HTTP proxy to use for connections
  • update update package [CHOICES: "yes","no"]

Usage:

    opkg.present("strace")
     update: "yes"
absent(force_depends, force_remove, autoremove, force_removal_of_dependent_packages)
Uninstall a package.

Subject:

    package

Aliases:

  • removed
  • remove

Parameters:

  • force_depends remove despite failed dependencies [CHOICES: "yes","no"]
  • force_remove Remove packages even if prerm hook fails [CHOICES: "yes","no"]
  • autoremove Remove packages that were installed to satisfy dependencies [CHOICES: "yes","no"]
  • force_removal_of_dependent_packages Remove package and all dependencies [CHOICES: "yes","no"]

Usage:

    opkg.absent("ncurses")
     force_remove: "yes"
